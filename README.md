# CF Domain Scanner

Инструмент для поиска Cloudflare-доменов, пригодных для **CDN VLESS fronting** в условиях российского ТСПУ.

Скрипт находит домены, через которые трафик проходит без блокировки — их можно использовать как `sni` и `host` в конфигурации VLESS/Xray с транспортом `ws` или `grpc` поверх Cloudflare.

---

## Как это работает

Каждый домен проходит три последовательных проверки:

**1. GET /** — базовая проверка:
- Домен резолвится в CF IP через Quad9 (обходит подмену DNS провайдером)
- В ответе есть `CF-Ray` заголовок или `Server: cloudflare`
- HTTP статус < 500
- Нет CF JS Challenge (сайт не требует решения капчи)
- Нет редиректа на блок-страницу провайдера (`warning.rt.ru`, `rkn.gov.ru` и др.)
- Ответ пришёл за 3 секунды

**2. X-Pad keepalive** — детект ТСПУ:
- Серия HEAD-запросов через одно keepalive соединение
- Каждый запрос добавляет ~4 КБ мусора в заголовок `X-Pad`
- Суммарно ~72 КБ — заведомо больше диапазона ТСПУ (1–69 КБ)
- Если соединение обрывается в этом диапазоне — домен заблокирован ТСПУ
- Метод работает независимо от размера сайта (даже если сайт отдаёт 403 с 3 КБ)

**3. TLS** — детект MITM и SNI-блокировок:
- Новое соединение с реальной проверкой сертификата
- Детектирует подмену сертификата провайдером (MITM)
- Детектирует SNI-блокировку
- Детектирует блокировку по версии TLS

Домен считается **живым** только если прошёл все три этапа.

---

## Требования

- Python 3.10+
- Запуск **с домашней машины в РФ** (под ТСПУ) — только тогда проверка имеет смысл. С VPS за рубежом ТСПУ не виден и результаты будут неточными.

## Установка

```bash
pip install aiohttp aiodns httpx
```

Или если `pip` ругается на систему:

```bash
pip install aiohttp aiodns httpx --break-system-packages
```

На Windows убедитесь что Python добавлен в PATH. Проверить:

```
python --version
```

## Использование

### Проверить один домен

```bash
python cf_scanner.py check --domain example.com
```

Выводит подробный отчёт по всем трём этапам проверки:

```
────────────────────────────────────────────────────────────
Домен:        example.com
IP:           104.21.44.100
CF IP:        ✓ Да
CF определён: ray
CF-Ray:       abc123-SVO
Server:       cloudflare
HTTP статус:  200
Байт получено:12800 (12 КБ)
Время:        0.41s

── Результаты проверок ──
  GET /       ✓ CF определён (ray), HTTP 200
  X-Pad       ✓ ТСПУ не обнаружен
  TLS         ✓ Сертификат валиден, нет MITM/SNI-блок

── Итог ──
  ✓ ЖИВОЙ — домен подходит для CDN VLESS fronting
────────────────────────────────────────────────────────────
```

---

### Сканировать топ-лист

Скачивает топ-1М доменов и проверяет случайную выборку:

```bash
python cf_scanner.py tranco --limit 5000 --random
```

Топ-5000 (не случайные, а первые из списка):

```bash
python cf_scanner.py tranco --limit 5000
```

Другой источник (umbrella, majestic, crux):

```bash
python cf_scanner.py tranco --source umbrella --limit 10000 --random
```

Первый запуск скачает список (~5–10 МБ). Последующие запуски используют кеш (`cache_tranco.csv` рядом со скриптом) и не качают заново, пока кеш не устареет (24 часа).

---

### Проверить свой список доменов

```bash
python cf_scanner.py file --file mylist.txt
```

Случайная выборка из файла:

```bash
python cf_scanner.py file --file mylist.txt --limit 500 --random
```

Формат файла — один домен на строку, строки начинающиеся с `#` игнорируются:

```
# мой список CF доменов
example.com
static.canva.com
assets.example.org
```

## Все параметры

### `check`

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--domain` | обязателен | Домен для проверки |
| `--dns` | `9.9.9.9` | DNS сервер для резолвинга |
| `--skip-tls` | выкл | Пропустить TLS-проверку (быстрее, не детектирует MITM) |

### `tranco`

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--source` | `tranco` | Источник: `tranco`, `umbrella`, `majestic`, `crux` |
| `--limit` | `10000` | Сколько доменов проверить |
| `--random` | выкл | Случайная выборка вместо топа |
| `--concurrency` | `30` | Параллельных проверок одновременно |
| `--output` | `results.txt` | Имя выходного файла (без расширения или `.txt`) |
| `--dns` | `9.9.9.9` | DNS сервер |
| `--skip-tls` | выкл | Пропустить TLS-проверку |

### `file`

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--file` | обязателен | Путь к файлу со списком доменов |
| `--limit` | нет | Ограничить количество доменов |
| `--random` | выкл | Случайная выборка |
| `--concurrency` | `30` | Параллельных проверок |
| `--output` | `results.txt` | Имя выходного файла |
| `--dns` | `9.9.9.9` | DNS сервер |
| `--skip-tls` | выкл | Пропустить TLS-проверку |

## Выходные файлы

После сканирования создаются два файла:

**`results.txt`** — готовый список живых доменов, по одному на строку:
```
example.com
static.canva.com
assets.cloudflare-site.com
```

**`results.log`** — детали по каждому найденному домену:
```
[2026-03-06 14:23:01] example.com          ip=104.21.44.100    status=200 bytes=12KB elapsed=0.41s cf=ray xpad=ok tls=ok
[2026-03-06 14:23:04] static.canva.com     ip=104.18.32.7      status=403 bytes=3KB  elapsed=0.28s cf=server xpad=ok tls=ok
```

Файлы перезаписываются атомарно через `.tmp` — при Ctrl+C данные не теряются.

---

## Консольный вывод

Во время сканирования живые домены выводятся сразу при нахождении:

```
✓ example.com                                 (200, 12KB, 0.4s, ray)
✓ static.canva.com                            (403, 3KB, 0.3s, server)
[14:23:05] Прогресс: 100/5000, живых CF: 7
✓ assets.some-site.com                        (200, 89KB, 1.1s, ray)
```

Мёртвые домены в консоль не выводятся — только прогресс каждые 100 доменов.

## Коды причин отказа (в .log файле)

| Код | Значение |
|---|---|
| `not_cf_ip` | IP не входит в диапазоны Cloudflare |
| `not_cf` | Нет CF-Ray и нет Server: cloudflare |
| `dns_fail` | Домен не резолвится |
| `http_5xx` | Сервер вернул 5xx ошибку |
| `http_451` | HTTP 451 — заблокировано по закону |
| `cf_challenge` | CF вернул JS Challenge (капча) |
| `isp_block_redirect` | Редирект на блок-страницу провайдера |
| `isp_block_body` | Блок-страница провайдера в теле ответа |
| `empty_response` | Сервер ничего не вернул |
| `timeout:NKB` | Таймаут, успело прийти N КБ |
| `tspu_cut:NKB` | Таймаут в диапазоне ТСПУ (1–69 КБ) |
| `tspu_rst:NKB` | RST-обрыв в диапазоне ТСПУ (1–69 КБ) |
| `rst:NKB` | TCP RST вне диапазона ТСПУ |
| `abort` | TCP соединение прервано |
| `refused` | Соединение отклонено |
| `tspu_slow:NKB` | ТСПУ замедляет (аномально долгий ответ) |
| `xpad_init:...` | Ошибка инициализации X-Pad теста |
| `tls_mitm` | Подмена сертификата провайдером (MITM) |
| `tls_mitm_selfsigned` | Самоподписанный сертификат (MITM) |
| `tls_mitm_unknown_ca` | Неизвестный CA (MITM) |
| `tls_mitm_hostname` | Несовпадение hostname (MITM) |
| `sni_block` | Блокировка по SNI |
| `tls_handshake_block` | Блокировка TLS handshake |
| `tls_dpi_eof` | DPI обрывает TLS соединение |
| `tls_cert_expired` | Истёкший сертификат |
| `tls_timeout` | Таймаут TLS соединения |
| `tls_rst` | RST при TLS соединении |

## Советы

**Оптимальный запуск для поиска доменов:**
```bash
python cf_scanner.py tranco --limit 10000 --random --concurrency 30
```
Случайная выборка лучше топа — топовые домены (google.com, youtube.com) часто заблокированы или под капчей. Середина списка содержит много полезных CF-доменов.

**Если нужно больше результатов:**
```bash
python cf_scanner.py tranco --source umbrella --limit 20000 --random
python cf_scanner.py tranco --source majestic --limit 20000 --random
```
Разные источники дают разные домены — можно комбинировать.

**Параллельность:**
Дефолт `--concurrency 30` — безопасное значение. При слабом интернете снизьте до 10–15. При хорошем канале можно поднять до 50, но X-Pad тест создаёт нагрузку на канал (каждая параллельная проверка может слать до 72 КБ).

**`--skip-tls` когда использовать:**
Если нужно быстро прогнать большой список и MITM у провайдера маловероятен. Ускоряет каждую проверку примерно на 0.5–1 сек.

**Кеш:**
Файлы `cache_tranco.csv`, `cache_umbrella.csv` и т.д. создаются рядом со скриптом. Можно удалить для принудительного обновления.
