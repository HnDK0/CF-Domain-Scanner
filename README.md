# CF Domain Scanner

Инструмент для поиска доменов, проксированных через Cloudflare, доступных из РФ.
Используется для настройки CDN fronting в VLESS/Xray конфигурациях.

## Зачем это нужно

Роскомнадзор блокирует домены, но не может заблокировать IP Cloudflare целиком — за ними стоят миллионы легальных сайтов. CDN fronting позволяет подключаться к серверу через чужой CF-домен: трафик выглядит как обращение к разрешённому сайту, а CF перенаправляет его дальше.

Проблема в том, что не все CF-домены одинаково доступны. Этот сканер автоматически находит те, которые работают.

## Установка

```
pip install aiohttp
```

Python 3.10+, работает на Windows и Linux.

## Режимы работы

### 1. `check` — проверить один домен

Быстро проверяет, стоит ли домен за CF и отвечает ли он.

```
python cf_scanner.py check --domain example.com
```

Вывод:
```
──────────────────────────────────────────────────
Домен:        example.com
IP:           104.21.55.23
CF IP:        Да
CF заголовки: Да
Server:       cloudflare
CF-Ray:       8a3f1c2d4e5b6789-FRA
HTTP статус:  200
Доступен:     Да
Итог:         CLOUDFLARE (подходит)
──────────────────────────────────────────────────
```

---

### 2. `tranco` — сканировать топ-лист доменов

Скачивает список самых популярных сайтов (Tranco или Cisco Umbrella) и проверяет каждый на CF + доступность. Топовые домены крупных компаний часто на CF и не заблокированы — это самый быстрый способ найти рабочие варианты.

```
# Первые 5000 доменов из Tranco
python cf_scanner.py tranco --limit 5000

# Первые 10000 из Umbrella, 150 параллельных запросов
python cf_scanner.py tranco --source umbrella --limit 10000 --concurrency 150

# Сохранить в свой файл
python cf_scanner.py tranco --limit 5000 --output my_cf_domains.json
```

Параметры:
- `--source` — `tranco` (по умолчанию) или `umbrella`
- `--limit` — сколько доменов проверять (default: 10000)
- `--concurrency` — параллельных запросов (default: 100, на слабом канале снизить до 50)
- `--output` — файл результатов (default: `results_tranco.json`)

Списки весят ~10MB, скачиваются автоматически.

---

### 3. `subdomain` — найти поддомены домена

Если нашёл рабочий CF-домен, этот режим ищет все его поддомены — они могут вести себя иначе и быть доступнее.

Источники поддоменов:
- **crt.sh** — Certificate Transparency logs, все домены которые когда-либо получали TLS сертификат
- **HackerTarget Reverse IP** — все домены на том же CF IP (может быть тысячи)
- **RapidDNS** — fallback если HackerTarget вернул ошибку лимита

```
python cf_scanner.py subdomain --domain example.com

# Больше параллельности
python cf_scanner.py subdomain --domain example.com --concurrency 100
```

Параметры:
- `--domain` — базовый домен (обязательно)
- `--concurrency` — параллельных запросов (default: 50)
- `--output` — файл результатов (default: `results_subdomains.json`)

> **Примечание:** HackerTarget бесплатный, но имеет лимит ~100 запросов в день без API ключа. Если вернул ошибку лимита — автоматически переключается на RapidDNS.

---

### 4. `cfip` — сканировать IP-подсети Cloudflare

Перебирает IP адреса из официальных CF-диапазонов, делает reverse DNS (PTR-запись) и проверяет найденные домены. Медленнее, но находит домены которых нет в топ-листах.

```
# Конкретный диапазон
python cf_scanner.py cfip --range 104.16.0.0/20 --limit 500

# Интерактивный выбор диапазона
python cf_scanner.py cfip
```

Без `--range` покажет список всех CF-диапазонов с размерами:
```
Доступные CF IP-диапазоны:
  [ 0] 103.21.244.0/22        (     1,022 адресов)
  [ 1] 103.22.200.0/22        (     1,022 адресов)
  ...
  [ 3] 104.16.0.0/13          (   524,286 адресов)
  [ 4] 104.24.0.0/14          (   262,142 адресов)

Введите номер или свой CIDR:
```

Параметры:
- `--range` — CIDR диапазон (опционально, без него — интерактивно)
- `--limit` — максимум IP для проверки (default: 1000)
- `--concurrency` — параллельность (default: 50)
- `--output` — файл результатов (default: `results_cfip.json`)

> PTR-записи есть не у всех IP, у CF это особенно редко. Режим полезен для разведки, но не как основной.

---

### 5. `file` — проверить свой список доменов

Принимает текстовый файл, один домен на строку. Строки начинающиеся с `#` — комментарии.

```
python cf_scanner.py file --file mylist.txt
python cf_scanner.py file --file mylist.txt --concurrency 200 --output found.json
```

`mylist.txt`:
```
# мои кандидаты
cloudflare.com
workers.dev
pages.dev
discord.com
```

## Формат результатов

Все режимы сохраняют JSON файл:

```json
{
  "total_checked": 5000,
  "cf_accessible": 312,
  "domains": [
    {
      "domain": "example.com",
      "ip": "104.21.55.23",
      "cf_ip": true,
      "cf_headers": true,
      "http_ok": true,
      "status": 200,
      "server": "cloudflare",
      "cf_ray": "8a3f...-FRA",
      "is_cf": true
    }
  ]
}
```

В `domains` попадают только домены где `is_cf=true` и `http_ok=true`.

Чтобы быстро извлечь список доменов из результата:

```bash
# Linux
cat results_tranco.json | python3 -c "import json,sys; [print(d['domain']) for d in json.load(sys.stdin)['domains']]"

# Windows PowerShell
(Get-Content results_tranco.json | ConvertFrom-Json).domains | ForEach-Object { $_.domain }
```

## Как определяется Cloudflare

Домен считается CF если выполняется хотя бы одно условие:
1. **CF IP** — IP адрес домена входит в официальные диапазоны Cloudflare (`/ips-v4`)
2. **CF заголовки** — ответ содержит `cf-ray` заголовок или `server: cloudflare`

Доступность: HTTP статус < 500 (200, 301, 302, 403, 404 — всё считается доступным, главное что сервер отвечает).

## Рекомендуемый порядок действий

1. **Найти кандидатов** — запустить `tranco --limit 5000`, получить список CF-доменов
2. **Расширить список** — для каждого найденного домена запустить `subdomain`, найти поддомены
3. **Проверить доступность из РФ** — скanner проверяет доступность с вашей машины. Если запускаете не из РФ — нужен прокси или проверяйте через сервисы типа `check-host.net`
4. **Использовать в конфиге** — найденные домены прописать как `host` в VLESS/Xray настройках

## Параметр concurrency

| Ситуация | Рекомендация |
|---|---|
| Слабый интернет / мобильный | 30–50 |
| Обычный домашний | 100–150 |
| Хороший канал / сервер | 200–300 |
| VPS с хорошим каналом | 300–500 |

При слишком высоком значении начнут падать таймауты и результаты будут неточными.
